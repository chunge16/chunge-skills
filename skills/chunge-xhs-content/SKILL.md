# Chunge XHS Content

Write publish-ready Xiaohongshu copy in one pass. Keep the output short, direct, easy to scan, and optimized for posting instead of long-form reading.

This skill is adapted from the content-structuring mindset behind `baoyu-image-cards`, but produces text only: title, body, and hashtags.

It can work in two modes:

- text-first: write from topic notes, product info, or rough points
- image-first: infer the best matching copy from reference images, image cards, or visuals generated by `baoyu-image-cards`
- combo mode: coordinate with `baoyu-image-cards` to produce a full Xiaohongshu image-and-copy package

## Core Goal

Optimize for three things at once:

- easy to understand at a glance
- strong enough to make people stop and read
- easy to copy and paste directly into Xiaohongshu

## Output Contract

Always produce these three blocks in order:

1. `标题`
2. `正文`
3. `话题`

Keep all three blocks in Chinese unless the user explicitly asks for another language.

## Copy-Safe Delivery

Treat copyability as a hard requirement. The user should be able to select the final output and paste it directly into Xiaohongshu with minimal cleanup.

- Output the final deliverable as plain text, not as Markdown decoration.
- The `正文` block must be plain text lines only.
- Do not use code fences.
- Do not add bullets like `-` or `*` unless the user explicitly asks for list formatting.
- Keep blank lines intentional and minimal so paste formatting stays clean.
- Do not insert analysis, notes, or option labels inside the copy unless the user asked for variants.
- When giving one version, send only one final copy block.
- When giving multiple versions, separate them clearly and keep each version independently copyable.

## Default Writing Rules

- Make the copy feel native to Xiaohongshu: vivid, direct, useful, and easy to skim.
- Keep the title within 20 Chinese characters when possible.
- Start the title with 1-2 fitting icons. Do not overload icons.
- Keep the body short. Default to 3-5 lines.
- Start each body line with a fitting icon or bullet-style emoji.
- Keep each line focused on one idea only.
- Prefer concrete wording over abstract claims.
- Use simple spoken Chinese. Avoid jargon, lecture tone, and bloated transitions.
- End with a light action cue only when it fits, such as inviting saving, trying, or commenting.
- Generate 5-8 hashtags by default. Keep them relevant, readable, and non-spammy.

## Workflow

### 1. Identify the post intent

Classify the request before writing:

- `种草`: recommendation, review, benefit-driven sharing
- `经验`: tips, method, workflow, tutorial
- `避坑`: warnings, mistakes, comparison, lessons learned
- `观点`: opinion, insight, concise stance
- `日常`: lifestyle, feeling, personal sharing

If the user does not specify the intent, infer the best fit from the material.

### 1.5 Choose the source mode

Choose the strongest available source:

- if the user provides structured notes or product info, use `text-first`
- if the user provides images, screenshots, covers, or image cards, use `image-first`
- if the user asks for both images and copy in one request, use `baoyu-image-cards` to shape the visual direction, then write copy that matches the image style, layout, and likely audience

For image-driven copy rules, read:

- `references/image-to-copy.md`
- `references/image-workflows.md`
- `references/combo-playbook.md`

### 2. Choose the writing angle

Pick one dominant angle only:

- benefit first
- contrast first
- result first
- emotion first
- checklist first

Do not mix multiple angles unless the user explicitly asks for alternatives.

For title formulas and body patterns, read:

- `references/title-formulas.md`
- `references/body-patterns.md`
- `references/icon-guide.md`
- `references/hashtag-guide.md`
- `references/variant-modes.md`
- `references/guardrails.md`
- `references/domain-modes.md`
- `references/examples.md`

### 3. Compress the message

Before drafting, reduce the source into:

- one core takeaway
- up to three supporting points
- one optional action cue

If the user gives too much material, summarize aggressively. Xiaohongshu copy should feel sharp, not exhaustive.

### 4. Choose the intensity

Use one intensity mode by default:

- `克制`: clean, useful, less emotional
- `标准`: balanced, native, versatile
- `爆点`: stronger rhythm and hook, but still believable

If the user does not specify, use `标准`.

### 5. Draft the copy

Produce one polished version by default.

When the user asks for multiple choices, provide 3 variants max:

- `克制版`
- `种草版`
- `爆点版`

Keep the structure stable so the user can copy it directly.

Use this default final layout:

标题
[one line title]

正文
[line 1]
[line 2]
[line 3]

话题
[#tag1 #tag2 #tag3 ...]

## Formatting Rules

- Use plain text only unless the user explicitly asks for Markdown tables or code blocks.
- Output `正文` as plain text lines for direct copying.
- Do not wrap the final copy in code fences, block quotes, or quotation marks.
- Do not add explanations before the deliverable unless the user asks for analysis.
- Avoid extra labels, numbering, or commentary that would break direct paste usage.
- Avoid excessive full-width punctuation like `！！！` or `～～`.
- Avoid clickbait that feels fake, exaggerated, or misleading.
- Avoid banned or risky claims in health, finance, or medical topics. Stay general and cautious.

## Adaptation Rules

- If the user provides a product, emphasize experience, use case, and concrete payoff.
- If the user provides notes or key points, reorganize them into a cleaner post instead of preserving their order.
- If the user provides images, infer topic, tone, audience, and posting angle from the visuals before drafting.
- If the user provides image cards, write copy that complements the cards instead of repeating every line already visible on the images.
- If the images already contain dense text, make the body lighter and more connective.
- If the images are emotionally strong or highly visual, let the title and first body line echo that mood.
- If the user asks for a stronger viral feel, increase punch and rhythm, but keep the wording believable.
- If the user asks for a calmer tone, reduce icons and emotional adjectives first.
- If the user asks for direct publish format, output only the final copy block.
- If the topic clearly belongs to a known content domain, adapt tone, icon choice, title rhythm, and hashtag style using `references/domain-modes.md`.
- If the request includes both content and image generation, treat `baoyu-image-cards` as the visual companion skill and make the copy feel aligned with the selected style and layout.
- If the user also needs cover text or image-card copy, suggest `baoyu-image-cards` after finishing the text task.

## Final Check

Before sending the result, verify:

- the title has a clear hook
- the body is short and readable on mobile
- each line adds new information
- the icons match the tone
- the hashtags match the topic and are not repetitive
- the final output can be copied without cleanup
- the tone matches the domain instead of sounding generic
